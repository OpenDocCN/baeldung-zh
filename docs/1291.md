## **1。Spring 和 Java**

#### **[> >模块系统的状态](https://web.archive.org/web/20220815044358/https://openjdk.java.net/projects/jigsaw/spec/sotms/)**【java.net】

非常好的介绍——不—**参考介绍**即将成为 Java 模块系统。好东西。

这里有一个模块系统的快速外观。

**[> >流表演](https://web.archive.org/web/20220815044358/http://blog.codefx.org/java/stream-performance/)**【codefx.org】

非常有趣的**流性能数据**，根据您系统的性能目标，可能还有可操作的数据。

#### **[> >如何使用 Java 8 函数式编程生成一个字母序列](https://web.archive.org/web/20220815044358/http://blog.jooq.org/2015/09/09/how-to-use-java-8-functional-programming-to-generate-an-alphabetic-sequence/)**【jooq.org】

好事降临到那些**使用 Java 8 streams** 的人身上，但只有那些使用 Clojure 的人留下的东西。

#### **[> >春季数据发布小鹅有什么新消息？](https://web.archive.org/web/20220815044358/https://spring.io/blog/2015/09/04/what-s-new-in-spring-data-release-gosling)**spring . io

此版本中的大量改进；我最兴奋的是新的 Querydsl 网络支持和 HAL 浏览器——这些看起来非常有用。

#### **[> >事件源和命令-查询职责分离简介](https://web.archive.org/web/20220815044358/http://squirrel.pl/blog/2015/08/31/introduction-to-event-sourcing-and-command-query-responsibility-segregation/)** [ 松鼠. pl

在一篇文章中同时介绍 CQRS 和事件采购并不容易，但这绝对是一篇很好的开篇文章，我希望这将是一个很长的系列文章。

#### **[> > 7 大发布后 Java 性能指标看点](https://web.archive.org/web/20220815044358/http://blog.takipi.com/7-java-performance-metrics-to-watch-after-a-major-release/)**【takipi.com】

一组可靠的**基本指标**，你确实需要一直跟踪，尤其是在将一个主要版本投入生产后。
