## **1。Java 和 Spring**

#### [**> >甲骨文宣布更多 Java 9 新特性！**](https://web.archive.org/web/20220521225510/http://jaxenter.com/new-java-9-features-announced-112654.html)

本周宣布了 Java 9 的新特性。我们知道模块化是主要的焦点，但是我们不知道**日志**正在翻新，**GC 选项**正在清理。

#### **[>>Java 时区处理入门](https://web.archive.org/web/20220521225510/http://vladmihalcea.com/2014/11/17/a-beginners-guide-to-java-time-zone-handling/)**

Java 中**处理时间的基础——好东西。**

#### **[> >中断执行者任务](https://web.archive.org/web/20220521225510/http://techblog.bozho.net/interrupting-executor-tasks/)**

我喜欢这个博客——它几乎总是我阅读后发现书签的东西。这是关于如何处理线程内部发生的**取消正在进行的工作的细节。**

我对几年前这样做只有模糊的记忆——不幸的是我没有做笔记。原来我可以直接引用这首曲子。

#### **[>>ExecutorService——10 个小窍门](https://web.archive.org/web/20220521225510/http://www.nurkiewicz.com/2014/11/executorservice-10-tips-and-tricks.html)**

当**使用线程池**时要遵循的实用建议。即使是隐藏在 Springs `@Async`抽象背后的人也会从这些要点中受益。总而言之，如果你正在使用它们，去读读这篇文章——你会为自己节省一些时间。

#### **[>>devo xx 2014–白板投票](https://web.archive.org/web/20220521225510/http://blog.joda.org/2014/11/devoxx-2014-whiteboard-votes.html)**

Devoxx 2014 投票者对 Java 生态系统的有趣快照。值得一读。

## **2。技术和思考**

#### **[> >连续交付 101:自动部署](https://web.archive.org/web/20220521225510/https://www.dynatrace.com/news/blog/continuous-delivery-101-automated-deployments/)**

**持续部署是我们要走的路**–我认为没有人会反对这一点。然而，到达那里是一个不同的故事——这是一个快速而中肯的 CD 成熟度模型——当你为下一个项目设置 Jenkins 时，这是一个非常有用的东西。

#### **[> >微服务架构中的测试策略](https://web.archive.org/web/20220521225510/http://martinfowler.com/articles/microservice-testing/)**

这种演示风格的资料是**开始了解微服务**的一个非常好的地方。

2014 年无疑是这种架构风格成长和采用的一年，但是——根据最近出现的**失败案例**的数量——它不应该被视为默认的首选。

无论如何，这是一个平衡的，放在一起的资源。

#### **[> >帽应拍](https://web.archive.org/web/20220521225510/http://damienkatz.net/2014/11/cap_should_be_clap.html)**

很好地阅读了**为什么延迟应该是我们关于 CAP** 的讨论以及我们关于可用性的推理的一部分。绝对值得一读。

#### **[>>](https://web.archive.org/web/20220521225510/http://www.daedtech.com/annihilating-complexity)**

事实上，有些人自然地给任何场景增加了复杂性，而有些人消除了复杂性，这是一个一旦你开始探索，你就会开始到处看到的事实。更糟糕的事情随处可见——比如数字 11(我认识一个人)。

长话短说——像所有模型一样，这个模型有点简化，但是非常有用。有用之处在于，它给了你一个很好的框架，让你可以分析性地看待自己对某个问题的解决方案，并加以改进。

#### **[> >作为一名技术主管，我还怎么写代码？](https://web.archive.org/web/20220521225510/https://www.thekua.com/atwork/2014/11/how-do-i-still-write-code-as-a-tech-lead/)**

关于**成为有效技术领导者的有用策略**。有效意味着不要在你和代码之间拉开距离——如果你停止编码，那么你就不再是一个好的领导者。
