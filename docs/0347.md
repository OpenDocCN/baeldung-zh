#### [>>](https://web.archive.org/web/20220521214228/http://blog.cleancoder.com/uncle-bob/2014/09/19/MicroServicesAndJars.html)

#### [> >微服务不是罐子](https://web.archive.org/web/20220521214228/http://www.giorgiosironi.com/2014/09/microservices-are-not-jars.html)

鲍勃大叔对微服务的价值提出了一个有趣但过于简单的观点，以及对这一观点的微妙回答。我看原文的第一反应是`hmm, he just found out about the microservice concept a few days ago and he's already writing about it?`

但后来，随着我不断思考，我意识到——作者可能是错的，也可能不是错的，但提出一个明确的立场既有价值又有点勇敢——我喜欢这一点。更重要的是，它肯定会打开对话的主题，不管最初的位置如何，这都是一件好事。

## 3。沉思

#### [> >帮我提高写作风格](https://web.archive.org/web/20220521214228/http://www.petrikainulainen.net/writing/help-me-to-improve-my-writing-style/)

关于写作风格的一个重要的批判性思考。当你的目标是教学和发展自己的声音时，沟通风格和方法非常重要——这就是为什么我说这篇文章“重要”。

我个人经历过我的“声音”的不同化身——我还远没有完成，所以这篇文章击中了我的要害。我也相信**是透明的，并且在公共场合思考**；这实际上是我开始 [Meta Baeldung](https://web.archive.org/web/20220521214228/http://meta.baeldung.com/) 的原因——在这个博客上记录我的实验，并通过把我的工作放在那里来学习。到目前为止，效果还不错。

#### [> >幸存下来的遗留代码与金主和采样](https://web.archive.org/web/20220521214228/http://blog.thecodewhisperer.com/2014/09/28/surviving-legacy-code-with-golden-master-and-sampling/)

另一篇关于使用遗留代码的战术文章。这不是我经常读到的关于遗留代码的“在更改代码之前将代码片段包装在测试中”的言论，而是实际的高级策略，远远不止于此。

代码密语是你需要订阅的网站之一。

#### [> >一年的博客](https://web.archive.org/web/20220521214228/http://vladmihalcea.com/2014/09/29/one-year-of-blogging/)

“教学是我的学习方式”——说得好。最大的误解之一是，你需要在开始教学之前成为一名专家。你真的没有；事实上，如果你等到你成为专家，你可能会远离那种感觉，那就是你的教学根本不会有共鸣。

我非常喜欢这些元作品——让我一瞥**构建自己的平台**背后的含义。很少有博客对这种信息足够透明，我觉得这是建立健康生态系统的重要组成部分。

另外——谢谢你提到 Vlad——祝你好运，把你的 [Hibernate masterclass](https://web.archive.org/web/20220521214228/http://vladmihalcea.com/tutorials/hibernate/) 放在一起。你显然对那方面有很深的理解，而真正能加速你学习的好课程在 Java 生态圈里少之又少。
