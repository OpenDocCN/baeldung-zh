2014 年初，我决定跟踪我的阅读习惯，并在 Baeldung 上分享最好的东西。

2014 年是不平凡的一年，每周都有回顾。我一直在做更多的阅读，以确保我涵盖和策划的东西是有价值的，实际上是有趣的。

如果你觉得我的评论有趣且有用，请在评论中告诉我。

## **1。Spring 和 Java**

#### **[> >资源服务器:Angular JS 和 Spring Security 第三部](https://web.archive.org/web/20220815050054/https://spring.io/blog/2015/01/20/the-resource-server-angular-js-and-spring-security-part-iii)T3】**

这是一个非常有用的系列文章的第三部分，深入探讨了 CORS 的本质，一个简单的令牌认证解决方案，当然还有 Spring Session。和前两部一样，这部也有很多值得借鉴的地方。

#### **[> >用 Docker 框搜索，弹性搜索，还有硒](https://web.archive.org/web/20220815050054/http://www.alexecollins.com/elastic-search-in-a-box-with-docker/)**

一篇 ATDD 驱动的、实用的、优雅简单的关于构建笑话搜索应用的文章。

我经常收到这个问题——`What's the best way for me to start learning X?`(Spring，Web 开发等)。这是一个不错的开始。

#### **[> > Fork/Join 框架 vs .并行流 vs. ExecutorService:终极 Fork/Join 基准](https://web.archive.org/web/20220815050054/http://blog.takipi.com/forkjoin-framework-vs-parallel-streams-vs-executorservice-the-ultimate-benchmark/)**

一个非常有趣的 Java 中并行化的各种**选项的分析/基准测试——一些值得深入研究的实数。**

## **2。技术和思考**

#### **[> >我们会需要它](https://web.archive.org/web/20220815050054/http://www.petrikainulainen.net/software-development/design/we-are-gonna-need-it)**

在匆忙摆脱瀑布的过程中，很有可能会走向另一个方向。这篇文章是对我们如何解决这个问题的思考。

#### **[> >互联网是伪去中心化](https://web.archive.org/web/20220815050054/http://techblog.bozho.net/the-internet-is-pseudo-decentralized/)**

一个关于网络拓扑和你如何开始游戏的思想实验——快速而有趣的阅读。

#### **[> >我将尝试定义科技债务](https://web.archive.org/web/20220815050054/http://www.daedtech.com/ill-take-a-crack-at-defining-tech-debt)**

这是对大家都熟悉的“技术债务”概念的一次很好的尝试。

#### **[>T5】开源服务发现](https://web.archive.org/web/20220815050054/http://jasonwilder.com/blog/2014/02/04/service-discovery-in-the-cloud/)T3】**

一旦**有了足够数量的服务**需要编排并部署到一个灵活的拓扑中，就有了让服务互相发现的问题。**静态配置将不再削减它。**

本文展示了解决这个特定问题的成熟的开源选项。

此外，本周我们发现 Spring 与这些解决方案之一——网飞的 Eureka: [通过 Spring Cloud 和网飞的 Eureka](https://web.archive.org/web/20220815050054/https://spring.io/blog/2015/01/20/microservice-registration-and-discovery-with-spring-cloud-and-netflix-s-eureka) 进行微服务注册和发现

#### **[> >从原始执念到领域造型](https://web.archive.org/web/20220815050054/http://blog.ploeh.dk/2015/01/19/from-primitive-obsession-to-domain-modelling/)**

有时，领域是放置它的正确位置。

#### **[> >将 Python 作为教学语言](https://web.archive.org/web/20220815050054/http://prog21.dadgum.com/203.html)**

我会把务实的思维带到我能找到的任何地方。当试图回答**`What's the best programming language a beginner should start with?`**这个问题时，这读起来既明智又务实

#### **[> >不破连胜也许](https://web.archive.org/web/20220815050054/http://zachholman.com/posts/streaks/)**

这是一篇有趣的文章，展示了众所周知的生产力建议“不要打破记录”是如何被应用到许多产品中的。然后盲目的去做只是因为它在那里。
