## **1。Spring 和 Java**

#### **[> >春季数据 JPA 教程:审计，第一部分](https://web.archive.org/web/20220117222325/http://www.petrikainulainen.net/programming/spring-framework/spring-data-jpa-tutorial-auditing-part-one/)**petrikainulainen

使用 Spring 数据和 JPA 的审计策略——这是任何成熟的生产级系统的重要的第一部分。

#### **[> >用 JBehave、PhantomJS 和 PageObjects](https://web.archive.org/web/20220117222325/https://blog.codecentric.de/en/2015/07/testing-web-applications-with-jbehave-phantomjs-and-pageobjects/)** [ 以代码为中心 ]测试 Web 应用

我总是发现页面对象模式在测试 web 应用程序时非常有用。你能得到的流畅的、类似英语的语法非常有用，这篇文章很好地说明了这一点。

Java 9 确实带来了许多很酷的新东西，将在未来几年推动语言和生态系统向前发展。

## **2。技术**

#### **[> >亚马逊 EC2 集装箱服务](https://web.archive.org/web/20220117222325/http://www.allthingsdistributed.com/2015/07/under-the-hood-of-the-amazon-ec2-container-service.html)**all things distributed

从 EC2 看**的新集装箱服务。非常有趣的东西，我目前看到的许多策略都是手动实现的。**

#### **[> >监控微服务:克服最大挑战的三种方法](https://web.archive.org/web/20220117222325/https://www.loggly.com/blog/monitoring-microservices-three-ways-to-overcome-the-biggest-challenges/)**loggly

关于微服务架构中的监控和警报的快速而有趣的阅读。

#### **[> >宁静被认为有害的](https://web.archive.org/web/20220117222325/http://www.nurkiewicz.com/2015/07/restful-considered-harmful.html)**nurkiewicz

坚持 RESTful 架构的一些缺点。

其中一些与 REST 本身无关，更多的是与它在野外实现的方式有关，但是这些要点中的大多数都有很好的借鉴之处，不管您是否同意它们。
