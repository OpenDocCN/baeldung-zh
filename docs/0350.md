2014 年初，我决定开始追踪我的阅读习惯，并在 Baeldung 上分享最好的东西。

管理我的阅读使它更有目的和多样化——我也希望通过让本周最好的内容上升到顶部来为你提供价值。

**开始了……**

## **1。Java**

#### **[> >了解 Spring Web 应用架构:经典方式](https://web.archive.org/web/20221208143837/http://www.petrikainulainen.net/software-development/design/understanding-spring-web-application-architecture-the-classic-way/)**

Petri 网新系列的开始——系统架构。这篇文章回顾了构建系统的经典方法。

#### **[> > Hibernate 应用级可重复读取](https://web.archive.org/web/20221208143837/http://vladmihalcea.com/2014/10/23/hibernate-application-level-repeatable-reads/)**

Hibernate Master Class 系列中的另一篇文章——这篇文章是关于具有可重复读隔离语义的**操作的。**

如果您正在使用 Hibernate 或将任何类型的数据写入关系数据库，理解这些核心概念是至关重要的。

#### **[> >让我们用 jOOλ](https://web.archive.org/web/20221208143837/http://blog.jooq.org/2014/10/23/lets-stream-a-map-in-java-8-with-joo%CE%BB/)** 在 Java 8 中流一张地图

简明扼要——在 Java 8 中流式传输地图应该更容易，现在确实如此。

## **2。弹簧**

#### **[> >看看如何用这个新教程构建、测试、保护和添加超媒体，“用 Spring 构建 REST 服务”](https://web.archive.org/web/20221208143837/https://spring.io/blog/2014/10/23/see-how-to-build-test-secure-and-add-hypermedia-with-this-new-tutorial-building-rest-services-with-spring)**

这个标题说明了一切，不是吗？

#### **[> >引入弹簧同步](https://web.archive.org/web/20221208143837/https://spring.io/blog/2014/10/22/introducing-spring-sync)**

介绍 Spring 生态系统的一个重要部分——**对补丁操作的支持**(灵感来自[的 Json 补丁规范](https://web.archive.org/web/20221208143837/https://tools.ietf.org/html/rfc6902))。

我可以看到——对于某些用例来说——这在客户端-服务器通信效率方面可能是一件大事。

## **3。技术和思考**

#### **[>>DSL 丛林](https://web.archive.org/web/20221208143837/http://techblog.bozho.net/?p=1559)T3】**

来自实际渠道的配置提示–我非常喜欢这篇文章，因为它非常实用。有人多久会说一次——看， **XML 在某些情况下确实很好**,不要马上抛弃它。

#### **[> >弹性搜索测试和 QA:通过随机测试运行增加覆盖率](https://web.archive.org/web/20221208143837/http://www.elasticsearch.org/blog/elasticsearch-testing-qa-increasing-coverage-randomizing-test-runs/)**

这种**测试策略**可以在几乎任何非平凡的系统中产生如此大的差异。我需要在我的测试中引入更多这种**随机输入**——这是肯定的。

#### **[>>](https://web.archive.org/web/20221208143837/http://martinfowler.com/bliki/SacrificialArchitecture.html)**

系统的架构自然向前发展。务实地用一个新的架构来取代它绝不是一个失败，而是一个我们需要意识到并明智地计划的事件。非常吸引人的阅读。

#### **[> >贵小区门口](https://web.archive.org/web/20221208143837/http://blog.codinghorror.com/your-community-door/)**

**建立一个社区绝非易事**——绝不是。该不该对不良行为严加管教？系统中有`mute`和`ignore`选项吗——这是个好主意吗？

我认为构建一个长期保持健康的社区需要很多经验(可能还有很多错误)，但这是一个非常有趣的问题。

#### [**> >敏捷:虚幻的希望和真实的承诺**](https://web.archive.org/web/20221208143837/http://www.daedtech.com/agile-false-hope-and-real-promise)

这里有一篇关于敏捷的文章，它尽最大努力不做判断，也不说鸡毛蒜皮的事情。
