2014 年初，我决定跟踪我的阅读习惯，并在 Baeldung 上分享最好的东西。

2014 年是不平凡的一年，每周都有回顾。我一直在做更多的阅读，以确保我涵盖和策划的东西是有价值的，实际上是有趣的。

如果你觉得我的评论有趣且有用，请在评论中告诉我。

## **1。Java 和 Spring**

#### **[> >引擎盖下有一个面向 Java 的增强泛型原型](https://web.archive.org/web/20220815032928/http://www.infoq.com/news/2014/12/Java10EnhancedGenerics)**

Java 10 中对泛型的改进。

不幸的是，看起来这些改进并没有完全具体化泛型，所以我们仍然有类型擦除，但是使用原语确实带来了一些有趣的可能性。

#### **[>](https://web.archive.org/web/20220815032928/http://www.infoq.com/interviews/java8-lambdas-aeron-montgomery)**

关于优化我们在 Java 中使用 Lambdas 的方式以及为什么我们应该关注它的有趣的 10 分钟讨论。

#### **[> > Java 8 WTF:模糊方法查找](https://web.archive.org/web/20220815032928/https://jvilk.com/blog/java-8-wtf-ambiguous-method-lookup/)**

Java 语言规范中的模糊性，以及在此之上的坚实而详细的分析。这是一篇值得你花时间去写的文章，就像接下来的文章一样。

**[> >带 CompletableFuture 的异步超时](https://web.archive.org/web/20220815032928/http://www.nurkiewicz.com/2014/12/asynchronous-timeouts-with.html)**

深入探讨如何利用 **Java 8 编写更好的并发代码**。

#### **[>>Spring Boot](https://web.archive.org/web/20220815032928/http://kielczewski.eu/2014/12/spring-boot-security-application/)安全应用**

这是一篇很长但很有用的参考文章，介绍了如何构建一个安全的 Boot web 应用程序。

#### **[> >春季重试——与您的项目集成的方式](https://web.archive.org/web/20220815032928/http://www.java-allandsundry.com/2014/12/spring-retry-ways-to-integrate-with.html)**

**重试失败的操作**是你在一个成熟的项目中需要做的事情之一。这个项目可能会有所帮助。

#### **[>弹簧框架 4 . 1 . 4&4 . 0 . 9&3 . 2 . 13](https://web.archive.org/web/20220815032928/https://spring.io/blog/2014/12/30/spring-framework-4-1-4-4-0-9-3-2-13-released)发布**

就在新年前，一个很酷的春季新版本，有很多错误修正。

## **2。技术和思考**

#### **[> >处理边缘案件](https://web.archive.org/web/20220815032928/http://techblog.bozho.net/handling-edge-cases-yes-uber/)**

仔细考虑你的应用程序的边缘情况，并很好地处理它们是成熟和尊重你的用户的标志，即使——是的——它确实给代码库增加了相当多的复杂性。

这并不意味着你需要在你的 V1 中这样做，也不意味着你会因为害怕不必要的边缘情况而发布一些有用的东西。

#### **[> >为什么中转重要](https://web.archive.org/web/20220815032928/https://www.mnot.net/blog/2014/12/27/why_intermediation_is_important)**

我们大多数人都在以某种方式或形式为网络开发——意识到潜在的“控制点”或我们可能认为理所当然的中介**会让我们做得更好**。

#### **[> >拯救一个项目和一家公司](https://web.archive.org/web/20220815032928/http://jacquesmattheij.com/saving-a-project-and-a-company)**

对咨询项目的深入**案例研究**，为顾问和企业主提供了许多有益的启示。这里有很多东西要学。
