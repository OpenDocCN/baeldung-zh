## **1。Java 和 Spring**

#### **[> > Java 不烂——你只是用错了](https://web.archive.org/web/20220525002359/http://www.jamesward.com/2014/12/03/java-doesnt-suck-youre-just-using-it-wrong)**

让我们全力以赴，从必读的**开始本周的复习。如果你正在阅读我的评论，你可能正在用 Java 做一些工作——在这种情况下，这篇文章将非常有用。**

其中一些观点是显而易见的，但我需要对那些不明显的观点做更多的阐述。

#### **[> >新 Java 版本——这不是 JDK 1.9](https://web.archive.org/web/20220525002359/http://www.infoq.com/news/2014/11/new-java-version-its-not-jdk19)**

我从没想过我会看到这一天，但看起来它真的可能会发生——**Java 走向语义版本化**。不再是精灵语`1.7.0_65-b20 (7u65)`——而是简单的 7.6.15。

既然我们在关注 JDK 相关新闻，[JDK 9 台的主页面](https://web.archive.org/web/20220525002359/https://openjdk.java.net/projects/jdk9/)刚刚更新了一些有趣的花絮。更多的 jep 以及一个关于**放弃 JSON API** 的提议！

#### **[> >重大发布后 Java 开发者应该使用的 15 个工具](https://web.archive.org/web/20220525002359/http://blog.takipi.com/15-tools-to-use-when-deploying-code-to-production/)**

每当我遇到一个系统**在没有监控或良好的日志解决方案**的情况下在生产中运行时——这种情况比你想象的更常见——我倾向于使用本文中的一些解决方案。

我总是发现这些高层次的比较很有用——它们给你提供了那些花时间为自己尝试所有选项的人所急需的背景。

#### **[> >终结者和幻影的致命缺陷](https://web.archive.org/web/20220525002359/http://www.infoq.com/articles/Fatal-Flaw-Finalizers-Phantoms)**

关于为什么不在 Java 中使用终结器的深入阅读。

#### **[> >最新天王集成改进春天](https://web.archive.org/web/20220525002359/https://spring.io/blog/2014/12/02/latest-jackson-integration-improvements-in-spring)**

我对最新春季版本中的新杰克逊美德感到非常兴奋——看起来它将在输出方面允许更多的灵活性。

#### **[> >春季数据发布的第一个里程碑火车福勒可用](https://web.archive.org/web/20220525002359/https://spring.io/blog/2014/12/01/first-milestone-of-spring-data-release-train-fowler-available)**

Spring 数据无需介绍——下面是下一个版本将提供的**。或者——如果您不打算在生产中使用它，那么通过使用这个里程碑可能已经得到了什么。我将很快尝试新的 Elasticsearch 版本。**

#### **[> > Spring 工具套件和 Groovy/Grails 工具套件 3.6.3 发布](https://web.archive.org/web/20220525002359/https://spring.io/blog/2014/12/02/spring-tool-suite-and-groovy-grails-tool-suite-3-6-3-released)**

站起来。升级您的 IDE。

#### **[> >避免不必要的元件扫描的弹簧配置](https://web.archive.org/web/20220525002359/http://lkrnac.net/blog/2014/10/avoid-component-scanning-of-spring-configuration/)**

如果您的 Spring 上下文中存在重复 beans 的问题，您可能甚至没有意识到这一点。这里有一个简单的 Spring 安全配置解决方案。

#### **[> >春季请求级记忆化](https://web.archive.org/web/20220525002359/http://vladmihalcea.com/2014/12/01/spring-request-level-memoization/)**

一个很酷的技术是利用缓存作为系统的横切关注点，并使用 Spring 在请求级别获得可重复的读取。

#### **[>>Spring rest 带链接资源的模板](https://web.archive.org/web/20220525002359/http://www.java-allandsundry.com/2014/11/spring-resttemplate-with-linked-resource.html)**

让哈尔意识到——真的很酷。
