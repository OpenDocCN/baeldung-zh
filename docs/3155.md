## **2。技术**

#### **[> >用同场 cookie](https://web.archive.org/web/20220812064225/https://www.igvita.com/2016/08/26/stop-cross-site-timing-attacks-with-samesite-cookies/)**【igvita.com】阻止跨站定时攻击

一个非常有前途的新草案，希望用一种新型的 cookie 来更新 RFC6265(主要的 HTTP 状态管理 RFC)。

如果被接受——这将对**减少一系列 CSRF 攻击和漏洞**有很大帮助。

这是一个非常令人兴奋的提议，并且在这篇文章中很好地解释了我们为什么需要它。

我认为现在我们都同意这样一个事实，即**代码审查是非常有益的**。当然，这并不能改变,**不容易掌握**的事实，尤其是在一个没有对新思想特别开放的文化的组织内部。

以我的经验来看，指标在这里很有帮助——当一个团队在一些关键指标上有一个不小的跳跃时，采用不再是需要“接受”的事情，而是成为团队内部的一个决定。

无论哪种方式，它们都经常发生——所以很高兴了解到有一家**公司实际上智能地存储了凭据数据**，所以当它发生时，也不是什么大事。
