## 1。Java 和 Spring

#### [> > Java 的过去、现在和未来](https://web.archive.org/web/20220521230257/http://www.infoq.com/presentations/java-history-present-future)

Java 生态系统像瑞典？对这句话有点困惑？是的——你需要听听 Brian Goetz 讲述 Java 语言、平台和生态系统是如何向前发展的。

如果有人能明智地谈论这个话题，那就是布莱恩·戈茨。

如果你是一名 Java 开发人员——我假设你是——理解平台是如何发展的，虽然不是立即可行，但仍然非常重要。

#### [> >企业 Java 中的事务隔离级别入门](https://web.archive.org/web/20220521230257/http://vladmihalcea.com/2014/12/23/a-beginners-guide-to-transaction-isolation-levels-in-enterprise-java/)

一个经过充分研究的实用的**事务隔离级别指南**——每个 java 开发人员都应该很好地理解它。即使你对主题有很深的理解，这也是一本好书。

#### [>如何封装春豆](https://web.archive.org/web/20220521230257/http://lkrnac.net/blog/2014/12/encapsulate-spring-bean/)

将您的 beans 包设为私有是一个非常好的实践，它将允许系统保持轻量级并被快速理解。默认情况下这样做是个好主意，只有在绝对需要时才允许 bean“逃离”它的包。

以及一些非常有用的**网络研讨会录音**:

*   **[> >从 0 到春安 4.0](https://web.archive.org/web/20220521230257/http://www.infoq.com/presentations/spring-4-security)**
*   **[>Java 上的 Spring 框架 8](https://web.archive.org/web/20220521230257/http://www.infoq.com/presentations/spring-java8)**

#### [> >展望 Java 9 货币与货币 API (JSR 354)](https://web.archive.org/web/20220521230257/http://www.mscharhag.com/2014/12/java-jsr-354-money-and-currency-api.html)

很好地介绍了绝对有趣的 JSR——**货币和货币 API** ,大量代码级示例深入参考实现。

## 2。技术和思考

#### [>>ThoughtWorks 雷达上的新奇有趣 2015 年 1 月](https://web.archive.org/web/20220521230257/http://www.infoq.com/news/2014/12/thoughtworks-radar-jan-2015)

ThoughtWorks Radar 始终是洞察和高度概括技术前景的良好来源。虽然它涵盖的大部分内容并不直接适用于您的工作，但适用的是值得一听的建议(查看他们对微服务的评论)。

#### [> >成为我们](https://web.archive.org/web/20220521230257/http://www.daedtech.com/be-the-we)

当我开始阅读这篇名字古怪的文章时，我不知道会发生什么。简而言之，这是一个坚实的建议，让你在进入一份新工作时有正确的心态。
