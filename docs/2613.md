2014 年初，我决定跟踪我的阅读习惯，并在 Baeldung 上分享最好的东西。

2014 年是不平凡的一年，每周都有回顾。我一直在做更多的阅读，以确保我涵盖和策划的东西是有价值的，实际上是有趣的。

## 1。Spring 和 Java

#### [>>API 网关模式:Angular JS 和 Spring 安全第四部分](https://web.archive.org/web/20220121030602/https://spring.io/blog/2015/01/28/the-api-gateway-pattern-angular-js-and-spring-security-part-iv)

**代理/网关**是构建 Spring webapp 时需要回答的核心问题之一。这有助于很多事情——CORS 就是其中之一。因此——在 Spring Central 上展开的 Angular 系列的新 Spring Security 的第四部分是非常有用的参考资料。

此外，我刚刚从我的文章待办事项列表中删除了一个主题，我一个字都不用写🙂

#### [> >休眠锁定模式——乐观锁定模式如何工作](https://web.archive.org/web/20220121030602/http://vladmihalcea.com/2015/01/26/hibernate-locking-patterns-how-does-optimistic-lock-mode-work/)

#### [> >阅读大行慢于 JDK 7、](https://web.archive.org/web/20220121030602/https://marxsoftware.blogspot.ro/2015/01/reading-large-lines-slower-in-jdk-7.html)8、

#### [> >使用 Spring Boot 登录到 Redis 并回退](https://web.archive.org/web/20220121030602/http://blog.florian-hopf.de/2015/01/logging-to-redis-using-spring-boot-and.html)

## 2。技术和思考

#### [> >避开完美设计](https://web.archive.org/web/20220121030602/http://www.daedtech.com/avoiding-the-perfect-design)

实用主义我们的行业缺少一个明确的“退出标准”。

**你会认为这是“年轻的枪”的错误，但它不是**。当你正在读这篇文章的时候，某个在这个行业已经工作了几十年的人正在交叉手指，签字同意`**The Big Rewrite**`。为什么？因为——`maybe this time…`

所以，也许不要那么做，也许去读读这本书。

#### [> >敏捷方法论还是敏捷软件？](https://web.archive.org/web/20220121030602/http://www.daedtech.com/agile-methodologies-or-agile-software)

我尽量远离 Baeldung 上的敏捷文章。这是因为我读到的关于这个主题的大部分材料都是有用的——有时。当然，除了一些小的例外——比如丹·诺斯(Dan North)之外——但他每隔一段时间就出版一次。不过，这本书值得一读。

#### [>>](https://web.archive.org/web/20220121030602/http://dandreamsofcoding.com/2015/01/23/taking-the-long-view/)

“长期游戏”是一种后天习得的技能，在你被烧伤一到多次之后。拥有采取必要步骤和长期跟踪结果的纪律可能是项目成功和失败的区别。

#### [> >所有的技术却没有一个爱情](https://web.archive.org/web/20220121030602/http://jacquesmattheij.com/all-of-the-tech-but-none-of-the-love)
