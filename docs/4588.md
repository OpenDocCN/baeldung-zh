## **2。技术和思考**

#### **[> >人配得上“什么”](https://web.archive.org/web/20220521222959/http://www.daedtech.com/people-deserve-the-what)**

关于如何授权给知识工作者的好建议，这样你会有长期的好结果。或者——如果你被委派——一个框架，你可以用它来了解你自己的情况，如果需要的话——改变它。

尤其是刚开始的时候，为了学习和成长进行优化是个不错的主意。

#### **[>因为读书是根本](https://web.archive.org/web/20220521222959/http://blog.codinghorror.com/because-reading-is-fundamental-2/)**

非常有趣，阅读如何通过在每个用户旁边显示有用的指标来鼓励社区中的有用行为，而不是不会带来好结果的指标。

#### **[> >用新技术做副业](https://web.archive.org/web/20220521222959/http://techblog.bozho.net/making-side-projects-with-new-technologies/)**

如果你正在阅读我的每周评论，那么你可能至少对你的工作有点热情——否则你不会在这里。现在——消费固然很好，但创造新事物——才是真正的学习。

这是一篇很好的快速阅读文章，讲述了为什么副业是一个好主意，以及它如何从长远来看肯定会让你受益。

#### **[> >为机器设计 API](https://web.archive.org/web/20220521222959/http://nordicapis.com/designing-apis-machines/)**

为其他开发者构建一个 API 并不是一件容易的事情。每个设计决策都需要仔细考虑，尤其是当 API 被其他人使用时。

现在——构建一个不是由开发者使用，而是由其他联网设备使用的 API 数量巨大——是另一种不同的困难。有见地的阅读看起来像什么。
