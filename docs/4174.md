## **1。Spring 和 Java**

#### **[> >便携、云就绪的 HTTP 会话](https://web.archive.org/web/20220625173043/https://spring.io/blog/2015/03/01/the-portable-cloud-ready-http-session)**

在 Spring Boot 之后，在所有正在进行的春季项目中，我最兴奋的是春季会议。这篇文章戏弄了一个中等罕见的 Redis 支持的 Http 会话，部署到引导。

#### **[>>JPA 和 Hibernate 级联类型入门](https://web.archive.org/web/20220625173043/http://vladmihalcea.com/2015/03/05/a-beginners-guide-to-jpa-and-hibernate-cascade-types/)**

理解 Hibernate 和 JPA 中级联的巨大资源——这里有很多例子值得探究。

#### **[>>Java Bootstrap:drowizard vs](https://web.archive.org/web/20220625173043/http://blog.takipi.com/java-bootstrap-dropwizard-vs-spring-boot/)**

对两种框架都是很好的介绍。

需要理解的重要一点是，Dropwizard 自 2011 年就已经存在，而 **Spring Boot 是一个更新的框架**。从这个角度来看，看到它在如此短时间内达到良好的采用和成熟是非常令人印象深刻的。

## **2。技术和思考**

#### **[> >离经叛道:抛弃你的 DVCS，回归理智](https://web.archive.org/web/20220625173043/http://bitquabit.com/post/unorthodocs-abandon-your-dvcs-and-return-to-sanity/)**

读起来很有趣。某人，某地**不认为`git`是蜜蜂的膝盖**？我说亵渎。

#### **[> >绩效考评简体](https://web.archive.org/web/20220625173043/http://www.daedtech.com/performance-reviews-simplified)**

在我的职业生涯中，每当我遇到绩效评估时，无论是在接受端还是在给予端，我总是会想

这篇文章围绕着同样的想法，提供了一个有趣、简单的替代方案——非常值得一读。

#### **[> >为什么不堵？](https://web.archive.org/web/20220625173043/http://techblog.bozho.net/why-non-blocking/)**

**非阻塞与阻塞架构**的一些非常有趣的考虑事项。

我也曾经使用过基于事件的系统，我可以肯定地看到它是如何引入了许多用传统架构根本不可能实现的酷东西，同时也使事情变得更加难以编写和推理。
