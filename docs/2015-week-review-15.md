## 1。Spring 和 Java

#### [>>Spring 如何实现与 Java 6、7、8 的兼容](https://web.archive.org/web/20220121021620/https://spring.io/blog/2015/04/03/how-spring-achieves-compatibility-with-java-6-7-and-8)

洞察智能构建支持 Java 8 的 Spring，但实际上不需要 Java 8。

对于任何在 Java 领域维护库/框架/API 的人来说，这里有一些要点。

#### [> >从战壕里跳出来:给配置豆注入属性值](https://web.archive.org/web/20220121021620/http://www.petrikainulainen.net/programming/spring-framework/spring-from-the-trenches-injecting-property-values-into-configuration-beans/)

如果你在你的项目中处理**许多属性，这绝对是一个值得遵循的可靠建议——使得管理所有这些复杂性变得容易得多。**

#### [> >结果集映射:基础知识](https://web.archive.org/web/20220121021620/http://www.thoughts-on-java.org/2015/04/result-set-mapping-basics.html)

看起来很有希望的系列文章的第一篇是关于**使用 JPA 和结果集**的。

#### [>>Hibernate 如何存储二级缓存条目](https://web.archive.org/web/20220121021620/http://vladmihalcea.com/2015/04/09/how-does-hibernate-store-second-level-cache-entries/)

**探索 Hibernate 缓存**乐趣和速度。从这件作品中可以学到很多东西。

#### [>>Spring Enable * annotation——编写自定义启用注释](https://web.archive.org/web/20220121021620/http://www.java-allandsundry.com/2015/04/spring-enable-annotation-writing-custom.html)

**一个`@EnableFoo`风格的注释**在复杂的系统中肯定能派上用场，没有理由仅仅局限于框架中的默认注释。

#### [>日志管理工具对决:Splunk vs. Logstash vs. Sumo 逻辑](https://web.archive.org/web/20220121021620/http://blog.takipi.com/log-management-tools-face-off-splunk-vs-logstash-vs-sumo-logic/)

即使对于中等规模的项目，日志管理、可见性、探索和分析也非常重要。在生产中实现适当的日志解决方案会让您受益匪浅。

这些工具在最初的几次中也为你节省了几个小时浏览基于文本的日志文件的时间。我最近经常用的是 ELK stack。
