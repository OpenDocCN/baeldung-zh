## **2。技术**

#### **[>>](https://web.archive.org/web/20220120083216/http://martinfowler.com/bliki/MicroservicePremium.html)**

微服务的宣传达到了新的高度——所以**知道何时使用这种相当复杂的架构风格，何时不使用**是至关重要的。这里有一个很好的指南来帮助我们导航决策过程。

#### **[> >在 AWS 上用 Graylog 收集日志](https://web.archive.org/web/20220120083216/http://techblog.bozho.net/log-collection-with-graylog-on-aws/)**

使用 Graylog 在 AWS 上设置日志收集系统的实用指南。

为此，我通常使用 Logstash 和 ELK 堆栈，但是 Graylog 看起来也是一个不错的选择。

不管具体的解决方案是什么，对于生产应用程序来说，拥有这样的系统是必须的。

## **3。沉思**

#### **[> >绥靖主义者、十字军，为什么会议通常会吸](https://web.archive.org/web/20220120083216/http://www.daedtech.com/appeasers-crusaders-and-why-meetings-usually-suck)**

这是一篇关于不同类型的性格和自然倾向在冲突时的自省文章。

当我很小的时候，我确实倾向于“十字军战士”,随着年龄的增长，我慢慢开始理解这是如何不理智和有效地利用我的时间。现在，如果我没有个人利益，我自然不会参与辩论——但有时我仍然不得不检查自己，意识到短暂的“T0”冲动。

有趣的读物。
