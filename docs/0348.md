# Baeldung 每周评论 40

> 原文：<https://web.archive.org/web/20220930061024/https://www.baeldung.com/2014-week-review-40>

2014 年初，我决定开始追踪我的阅读习惯，并在 Baeldung 上分享最好的东西。

管理我的阅读使它更有目的和多样化——我也希望通过让本周最好的内容上升到顶部来为你提供价值。

**开始了……**

## **1。Java 和 Spring**

#### **[> >用龙目](https://web.archive.org/web/20220521214624/http://www.mscharhag.com/2014/09/reduce-boilerplate-code-in-java-applications-with-project-lombok.html)** 项目减少 Java 应用程序中的样板代码

Lombok 项目的快速而有用的**介绍，以及使用它删除系统中一些样板文件的方法。**

我个人认为 Lombok 是一种“治标不治本”的方法——适度有用，但对于接手项目的新开发人员来说也是一个障碍。

#### **[> >你的代码覆盖率度量没有意义](https://web.archive.org/web/20220521214624/http://blog.frankel.ch/your-code-coverage-metric-is-not-meaningful)**

我认为这是一个众所周知的事实—**代码覆盖率并不能保证任何事情**,也绝对不应该成为企业关注的一个指标，至多是开发人员跟踪和使用的一个工具。

#### **[>>Java 8 Lambdas–引擎盖下的一瞥](https://web.archive.org/web/20220521214624/http://www.infoq.com/articles/Java-8-Lambdas-A-Peek-Under-the-Hood)**

Java Lambdas 有很多**介绍——这是其中较好的一个。**

#### **[>](https://web.archive.org/web/20220521214624/http://blog.codeleak.pl/2014/10/exposing-spring-data-repositories-over-rest.html)**

粗略地看一下 **Spring Data REST** 以及如何用 Spring Boot 设置它来实现一个简单的 API。很有用的快速落地跑。

#### **[> >流媒体 JSON 补丁从 Spring 到 React UI](https://web.archive.org/web/20220521214624/https://spring.io/blog/2014/10/08/streaming-json-patch-from-spring-to-a-react-ui)**

当一个框架探索网络的边缘并向前看时，这是一个好迹象。这篇简短的文章是对如何使用 JSON 补丁在 STOMP 协议上流式传输数据的初步介绍。

可能不会马上使用这个，但是，不管怎样，这是一个非常酷的练习。

## **2。技术和思考**

#### **[> >清理那些讨厌的垃圾抽屉](https://web.archive.org/web/20220521214624/http://blog.thecodewhisperer.com/2014/10/04/clean-up-those-nasty-junk-drawers/)**

关于如何获得更好的考试结构和分组的深入建议，这里有很多值得思考的东西。

#### **[>](https://web.archive.org/web/20220521214624/http://www.daedtech.com/chesstdd-18-removepiece-and-housekeeping)**

经过一段时间的休息后，国际象棋 TDD 系列的新装置。看看一个新的代码库(和 Java 代码库)对这个系列意味着什么将会很有趣——也许是倾向于 BDD 风格的惯例。

#### **[> >移动速度快&破空](https://web.archive.org/web/20220521214624/http://zachholman.com/talk/move-fast-break-nothing/)**

当系统在真正的用户手中时，你正在发展这个系统。如果你不是，你会是(这就是我们所做的全部，不是吗？).

既然我们已经确定了这一点——跳过去读读这篇文章——这不是废话。

#### **[> >最终信仰](https://web.archive.org/web/20220521214624/https://signalvnoise.com/posts/3776-faith-in-eventually)**

一个有趣的方式来**看技术深度**(以及其他)——来自 Jason Fried。

这绝对是一种习得的技能——知道现在要解决什么，以后要推动什么，并且能够接受事物持续不完美的状态。

#### **[> >我是如何策展人的](https://web.archive.org/web/20220521214624/http://www.mdswanson.com/blog/2014/10/03/how-i-curate-the-indyhackers-newsletter.html)**

这既是元的，也非常适合包含在时事通讯风格的文章中，如每周评论。这是一篇非常有趣的关于管理时事通讯的技术和实践的文章。

从今年年初开始，我就一直在这里写每周回顾，一路上我简化了我的流程，学到了很多最佳实践，但我还没有自动化任何东西。也许在 2015 年。
