# Baeldung 每周评论 11

> 原文：<https://web.archive.org/web/20220930061024/https://www.baeldung.com/2015-week-review-11>

2014 年初，我决定跟踪我的阅读习惯，并在 Baeldung 上分享最好的东西。

2014 年是不平凡的一年，每周都有回顾。我一直在做更多的阅读，以确保我涵盖和策划的东西有价值，实际上值得一读。

如果你觉得我的评论有趣且有用，请在评论中告诉我。

**开始了……**

## 1。Spring 和 Java

#### [>>Mark Reinhold on Java 9 and Beyond](https://web.archive.org/web/20210803043939/http://www.infoq.com/news/2015/03/java9-and-beyond)

Java 未来的**计划。**

#### [>>【JDK】8 个流和分组](https://web.archive.org/web/20210803043939/https://marxsoftware.blogspot.ro/2015/03/jdk-8-streams-and-grouping.html)

强大地使用流来**重组和分类后备集合**。

#### [> >冬眠级联型。锁定陷阱](https://web.archive.org/web/20210803043939/http://vladmihalcea.com/2015/03/12/hibernate-cascadetype-lock-gotchas/)

进一步深入探究休眠中**锁定的代码。**

#### [> >测试收集实施用番石榴](https://web.archive.org/web/20210803043939/http://blog.codefx.org/techniques/testing/test-collection-implementations-with-guava/)

使用 Guava 中已经存在的测试来测试定制集合实现的一种有趣方式。不知道番石榴可以这样做。

#### [> >与](https://web.archive.org/web/20210803043939/http://blog.frankel.ch/become-a-devops-with-spring-boot) 成为 DevOps

这是**监控 Spring Boot 应用**的一个非常好的起点。

#### [> >彼得·劳雷描述了 Pb 级 JVM](https://web.archive.org/web/20210803043939/http://www.infoq.com/news/2015/03/petabyte-jvms)

这是一个非常酷的 JVM 探索，具有非常大的堆(32GB+)。

#### [>>Activiti 和 Spring Boot](https://web.archive.org/web/20210803043939/https://spring.io/blog/2015/03/08/getting-started-with-activiti-and-spring-boot)入门

对 Spring Boot 生态系统内的活动的可靠、专注于代码的**介绍。我特别喜欢测试部分——这通常需要一点 API 探索才能做好。**

#### [> >入门:创建 Web 应用项目](https://web.archive.org/web/20210803043939/http://www.petrikainulainen.net/programming/gradle/getting-started-with-gradle-creating-a-web-application-project/)

虽然我个人不使用 Gradle，但如果我决定加入 Gradle 阵营，这个系列肯定会是我的首选。
