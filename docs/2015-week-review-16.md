## 1。Spring 和 Java

#### [> >从战壕中跃起:返回运行时配置为 JSON](https://web.archive.org/web/20220122021229/http://www.petrikainulainen.net/programming/spring-framework/spring-from-the-trenches-returning-runtime-configuration-as-json/)

让**我的应用程序使用的核心属性在启动时登录**是很重要的——它可以很容易地看到环境的确切布局并快速发现潜在的问题。

本文中介绍的简单而强大的机制使得设置起来非常简单，并且可以很容易地扩展到您的应用程序中可能使用的任何一类属性。

#### [> >结果集映射:复杂映射](https://web.archive.org/web/20220122021229/http://www.thoughts-on-java.org/2015/04/result-set-mapping-complex.html)

上周，本系列探讨了基本的映射——现在，它将深入探讨更高级的用例，例如**将一个查询结果映射到多个实体**。

#### [> >使用 Apache Kafka 进行集成，数据处理管道使用 Spring](https://web.archive.org/web/20220122021229/https://spring.io/blog/2015/04/15/using-apache-kafka-for-integration-and-data-processing-pipelines-with-spring)

## 2。技术

#### [> >吻与本质复杂](https://web.archive.org/web/20220122021229/https://techblog.bozho.net/kiss-with-essential-complexity/)

代码的复杂性——绝对值得一读。

我喜欢阅读那些以开发软件为生的人写的文章——**这类文章有一些务实和真实的东西。是的——你可以也应该让事情变得更好，但是不要期望你的项目中有教科书般简单的代码。**

#### [>微服务设计模式](https://web.archive.org/web/20220122021229/http://blog.arungupta.me/microservice-design-patterns/)

一篇扎实的文章，阐述了如何**设计微服务架构**。抛开宣传——这实际上是有用的。

#### [> >跳到企业缓存前要考虑的事情](https://web.archive.org/web/20220122021229/http://vladmihalcea.com/2015/04/16/things-to-consider-before-jumping-to-enterprise-caching/)

有一点要提醒的是，在没有首先了解你正在把什么潜在的数据一致性问题引入你的应用之前，不要启用缓存。处理这些问题肯定有方法——很好的方法，但重要的是首先要很好地理解这些是什么。

也值得一读:

*   #### [> > AWS cost: the five most common errors](https://web.archive.org/web/20220122021229/http://cloudacademy.com/blog/aws-costs-5-biggest-mistakes/)

*   #### [> > Brand-new G2 instance type, with 4 times higher GPU capability](https://web.archive.org/web/20220122021229/https://aws.amazon.com/blogs/aws/new-g2-instance-type-with-4x-more-gpu-power/)
