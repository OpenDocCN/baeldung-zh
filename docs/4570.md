## **1。Spring 和 Java**

#### **[> >弹簧框架 4.2](https://web.archive.org/web/20220121013052/https://spring.io/blog/2015/05/26/spring-framework-4-2-goes-rc1)**

是的， **Spring 4.2 在野外有了第一个候选发布版本**。

我感到兴奋的一件事是更好的活动支持。

#### **[> >清洗螺纹](https://web.archive.org/web/20220121013052/http://www.javaspecialists.eu/archive/Issue229.html)**

何时使用(以及何时避免)`ThreadLocal`，当然还有**如何清理**，确保线程返回到池中时，它没有抓住任何它不应该抓住的东西。

#### **[> >改造收藏](https://web.archive.org/web/20220121013052/http://blog.codefx.org/libfx/transforming-collections/)**

#### **[> >终极 JSON 库:JSON . simple vs GSON vs Jackson vs JSON](https://web.archive.org/web/20220121013052/http://blog.takipi.com/the-ultimate-json-library-json-simple-vs-gson-vs-jackson-vs-json/)**

## **2。技术**

#### **[>>Web API 中的下一级](https://web.archive.org/web/20220121013052/http://www.amundsen.com/blog/archives/1159)**

由 Mike Amundsen 编辑的关于当今 API 状态的 InfoQ 的一个新的、正在进行的系列。这将是一个有用的书签页，并回来。

#### **[>>](https://web.archive.org/web/20220121013052/http://martinfowler.com/bliki/Yagni.html)亚尼**

一个结构良好、细致入微且实用的**对雅格尼**的深入探究，充满了来自中土世界的例子。这绝对值得一读(或两读)。

#### **[>揭秘 CQRS](https://web.archive.org/web/20220121013052/https://blog.codecentric.de/en/2015/05/demystified-cqrs/)**

为了更好地理解 CQRS，这是一本必读的书——它是什么，为什么使用它是个好主意。

#### **[> >如何才能构建更好的复杂系统？集装箱，微服务，持续交付。](https://web.archive.org/web/20220121013052/http://highscalability.com/blog/2015/4/27/how-can-we-build-better-complex-systems-containers-microserv.html)**

关于复杂系统逻辑解耦的一些有趣说明。

也值得一读:

*   #### **[> > Ec2 instance history](https://web.archive.org/web/20220121013052/https://aws.amazon.com/blogs/aws/ec2-instance-history/)**

*   #### **[> > Eight different types of programmers](https://web.archive.org/web/20220121013052/http://david.elbe.me/programming/2015/05/25/different-types-of-programmers.html)**

*   #### **[> > [DDD] into one](https://web.archive.org/web/20220121013052/http://blog.sapiensworks.com/post/2015/05/25/DDD-Concepts-As-One-Liners)**

## **3。沉思**

#### **[> >如何让你的开计划办公室少吸](https://web.archive.org/web/20220121013052/http://www.mattblodgett.com/2015/05/open-plan-office-suck-less.html)**

有趣且相当准确的**图纸**展示了如何巧妙布置办公空间。

#### **[> >学习的七个阶段](https://web.archive.org/web/20220121013052/http://dandreamsofcoding.com/2015/05/26/seven-stages-of-learning/)**

这是一本非常有趣的读物，它揭示了学习新技能/语言的过程。我真的很喜欢学习阶段是如何以一种结构化的逻辑节奏呈现的。
