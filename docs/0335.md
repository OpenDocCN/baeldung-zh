## 1。Java 和 Spring

#### [> >为数据访问代码编写测试——单元测试是浪费](https://web.archive.org/web/20220521225147/http://www.petrikainulainen.net/programming/testing/writing-tests-for-data-access-code-unit-tests-are-waste/)

在开发人员发现 TDD 之后，过度使用单元测试是一个普遍的特征**——这是由于发现带来的热情和心态的转变。**

经过几年的勤奋实践，随着对假设的不断重新评估，同一个开发人员开始倾向于集成测试。

另一部分时间和**单元测试被用于它们的本意——一个设计工具**,而不是检查系统的正确性。

它在这儿停吗？我不知道——但我希望不知道。无论你在这条道路上的哪个地方，这篇文章都是一篇好文章，所以赶快行动起来阅读它吧。

#### [> > 6 个还没转到 Java 8 的理由](https://web.archive.org/web/20220521225147/http://www.takipiblog.com/2014/07/09/6-reasons-not-to-switch-to-java-8-just-yet/)

一些 Java 8 的注意事项。我想我们都同意，Java 8 是平台向前迈出的坚实一步，尤其是在经历了这么长时间的停滞之后，但没有什么是完美的——立即知道会发生什么是值得的。

#### [> > JUnit:用 Java 8 和 Lambda 表达式测试异常](https://web.archive.org/web/20220521225147/http://blog.codeleak.pl/2014/07/junit-testing-exception-with-java-8-and-lambda-expressions.html)

用 Java 8 构建的用于异常检查的 DSL——是的，JUnit 又变得很酷了。

#### [> >冬眠身份、序列和表(序列)生成器](https://web.archive.org/web/20220521225147/http://vladmihalcea.com/2014/07/08/hibernate-identity-sequence-and-table-sequence-generator/)

Vlad 正在开发他的 Hibernate 系列——这篇文章是一篇关于主键策略的很好的初级文章。

## 2。沉思

#### [> >通过分析避免瘫痪的秘诀](https://web.archive.org/web/20220521225147/http://www.daedtech.com/the-secret-to-avoiding-paralysis-by-analysis)

陷入杂草中，陷入“如果怎样”和**中，因此无法交付**是一种真正的耻辱。

TDD 是一种技术，它可以让你摆脱那种思维模式，进入一个关于你的系统和你的选择的渐进的和有机的框架。高度的实用主义也行。

#### [> >你最大的弱点是什么:雇主-应聘者阻抗不匹配](https://web.archive.org/web/20220521225147/http://www.daedtech.com/whats-your-greatest-weakness-the-employer-candidate-impedance-mismatch)
