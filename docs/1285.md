2014 年初，我决定跟踪我的阅读习惯，并在 Baeldung 上分享最好的东西。

## 1。Spring 和 Java

#### [> >发现线程被构建的位置](https://web.archive.org/web/20220126111718/http://www.javaspecialists.eu/archive/Issue226.html)

这是一个控制从创建线程的**的有趣例子——并展示了在这个过程中一些很酷的 Java 8 语法的使用。**

**[> >幂等与时态脱钩](https://web.archive.org/web/20220126111718/http://www.nurkiewicz.com/2015/02/journey-to-idempotency-and-temporal.html)**

如果你的工作以任何方式涉及到 HTTP，无论是哪种形式，这一条是**一条必须慢慢读**。这可能是一个有弹性的、成熟的系统和一个有时能正常工作的系统之间的差别。

#### [> >加盟弦在 JDK 8](https://web.archive.org/web/20220126111718/https://marxsoftware.blogspot.ro/2015/02/joining-strings-in-jdk-8.html)

我们不再需要使用 Guava 来连接字符串——Java 8 有一些有用的 API 可以很好地完成这项工作。

#### [> >休眠锁定模式——悲观 _ 读和悲观 _ 写如何工作](https://web.archive.org/web/20220126111718/http://vladmihalcea.com/2015/02/24/hibernate-locking-patterns-how-do-pessimistic_read-and-pessimistic_write-work/)

Vlad 继续他在**休眠悲观锁模式**中非常有用的尝试。

## 2。技术和思考

#### [> >经验报告:弱代码归属](https://web.archive.org/web/20220126111718/http://blog.jayfields.com/2015/02/experience-report-weak-code-ownership.html)

**对团队中的`code ownership`有意向**是很多团队没有迈出的重要的第一步。但是一旦明确考虑了这一点，就有了在`Collective`和`Weak`代码所有权之间的选择——我完全同意这篇文章——根据我的经验，弱所有权更好。

#### [**> > 10x 开发商，复议后**](https://web.archive.org/web/20220126111718/http://www.daedtech.com/10x-developer-reconsidered)

`10x Developer`的概念加上一点细微差别。你知道，就像现实生活一样。

#### [>>is-a vs .](https://web.archive.org/web/20220126111718/http://dandreamsofcoding.com/2015/02/23/is-a-vs-has-a/)

有趣的是，马斯洛需求层次理论为在现代公司工作**做了修改。**

#### [> >你要知道什么时候停止](https://web.archive.org/web/20220126111718/http://blog.thecodewhisperer.com/2015/02/15/you-have-to-know-when-to-stop/)

**TDD**的一个好处是，你知道**什么时候停止**，这个好处我没有考虑过，但我知道我会考虑，这个好处非常有意义。
