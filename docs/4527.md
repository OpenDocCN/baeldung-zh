2014 年初，我决定跟踪我的阅读习惯，并在 Baeldung 上分享最好的东西。

2014 年是不平凡的一年，每周都有回顾。我一直在做更多的阅读，以确保我涵盖和策划的东西有价值，实际上值得一读。

## **2。技术**

#### **[> >高级开发人员的角色](https://web.archive.org/web/20220121030505/http://mattbriggs.net/blog/2015/06/01/the-role-of-a-senior-developer/)**

这是一本关于在我们的行业中学习和工作的自省、深思熟虑的读物。

我会说这是本周的“必读书目”,如果本周没有这么多其他该死的好读物的话。比如下一个。

#### **[> >全然不顾饭桶犯下的历史](https://web.archive.org/web/20220121030505/http://zachholman.com/posts/git-commit-history/)**

使用 git 工作？然后读一读。我从这件事中学到了很多。

#### **[> >用静态分析提升自己的技术](https://web.archive.org/web/20220121030505/http://www.daedtech.com/improving-your-craft-with-static-analysis)**

一个实用的静态分析的原因介绍；如果你还没有利用像 PMD、Checkstyle 或 Findbugs 这样的工具，这是一个很好的开始动机。

#### **[>>](https://web.archive.org/web/20220121030505/http://martinfowler.com/bliki/MonolithFirst.html)**

如果你正在考虑在绿地项目中使用**微服务架构，或者甚至在你的视野中的任何地方使用微服务，这是一个很好的选择。**
