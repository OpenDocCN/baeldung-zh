## 1。Spring 和 Java

#### [> >反应式编程笔记第二部分:写一些代码](https://web.archive.org/web/20220812064952/https://spring.io/blog/2016/06/13/notes-on-reactive-programming-part-ii-writing-some-code) [ spring.io ]

**反应式系统是一种完全不同的系统**，非常适合一小组场景。

如果你想滑向冰球要去的地方，这是一个很好的记录。

#### [> >危机中的爪哇 EE 8](https://web.archive.org/web/20220812064952/http://www.xenonique.co.uk/blog/?p=2106)【xenonique.co.uk】

既然我们正在讨论我们的发展方向，那么 Java EE 8 的状态和前景对于整个 Java 社区和生态系统来说是非常重要的。

#### [> >如何用 Hibernate](https://web.archive.org/web/20220812064952/https://vladmihalcea.com/2016/06/13/how-to-implement-a-custom-string-based-sequence-identifier-generator-with-hibernate/)【vladmihalcea.com】实现一个自定义的基于字符串的序列标识符生成器

一个超级实用的，专注于用 Hibernate 生成字符串 id 的**解决方案(这实际上是我过去漫游过的东西)。**

和往常一样，Vlad 在社区的帮助和参与下选择了他的主题，这是一种非常可靠的方式。一个很酷的资源。

#### [> >如何正确实现 hashCode](https://web.archive.org/web/20220812064952/http://blog.codefx.org/java/implement-hashcode-correctly/)【codefx.org】

上周我们看了`equals`之后的下一篇回归基础的文章。我就知道会有这一天。

#### [> >配置一次，处处运行:解耦配置与运行时](https://web.archive.org/web/20220812064952/https://www.infoq.com/articles/Apache-Tamaya-Configure-Once-Run-Everywhere)【infoq.com】

**现实世界的项目配置**从来没有我们最初想象的那么简单。

现在，我不确定标准是否是这里的答案，但是这里的实用方法看起来很有趣。

#### [> >是时候忘记你所知道的关于 Java 异常的一切了](https://web.archive.org/web/20220812064952/http://blog.takipi.com/the-truth-behind-the-big-exceptions-lie/)【takipi.com】

这是一篇关于如何很好地处理异常的高水平文章。它读起来也不错，我认为亚历克斯写这个很有趣。

#### [> >你的下一个项目应该使用 JPA 吗？](https://web.archive.org/web/20220812064952/http://www.thoughts-on-java.org/use-jpa-next-project/)【thoughts-on-java.org】

Hibernate 和 JPA 当然不适合所有类型的项目，**但是它们是大多数项目的坚实基础。如果你真的很了解这个工具，你会惊讶你能走多远。**

当你开始一个新项目时，这种交互式的书面记录有助于你做出决定，或者至少给你一些关于这个决定的背景。
