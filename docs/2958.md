**开始了……**

## **1。Java 和 Spring**

#### **[> >一更库为 JDK 9？](https://web.archive.org/web/20220521210934/http://blog.joda.org/2014/11/one-more-library-for-jdk-9.html)**

一个“要么行动，要么闭嘴”的战斗口号，作为 JDK 号的补充——它可能小到足以进入 JDK 9 号，也属于那里。有什么想法吗？

#### **[> >从 Joda-Time 转换到 java.time](https://web.archive.org/web/20220521210934/http://blog.joda.org/2014/11/converting-from-joda-time-to-javatime.html)**

从我记事起，我就一直在使用`joda-time`。然而，进入 JDK 8 领域——是时候切换回标准日期库了。以下是方法。

#### **[> >一种实体造型的缩放策略](https://web.archive.org/web/20220521210934/https://vladmihalcea.com/an-entity-modeling-strategy-for-scaling-optimistic-locking/)**

另一篇关于乐观锁定的好文章——这次是关于对实体使用**版本的策略。这看起来像是一本正在制作中的小型电子书。**

#### **[>>Java 语言中没有属性](https://web.archive.org/web/20220521210934/http://blog.joda.org/2014/11/no-properties-in-java-language.html)**

`“The Java Language is not getting properties”`

#### **[> >永恒性如何帮助](https://web.archive.org/web/20220521210934/http://www.yegor256.com/2014/11/07/how-immutability-helps.html)**

很好地理解了为什么**不变性应该是库设计者的主要关注点**。对其他人来说确实如此，但是被许多开发人员使用的库的设计定下了基调，应该以身作则。

#### **[>>JMeter 2.12 版本已出](https://web.archive.org/web/20220521210934/https://jmeter.apache.org/changes.html)**

JMeter 是一个必不可少的工具，我发现自己在每个重要的项目中都在使用它。因此，当一个新版本推出了许多很酷的功能时，我很兴奋。

#### **[> >春季开机大战包装](https://web.archive.org/web/20220521210934/http://www.java-allandsundry.com/2014/11/spring-boot-war-packaging.html)**

简明扼要地讲述如何**为 Spring Boot 项目**进行 war 打包，而不是可执行的 jar 方法。
