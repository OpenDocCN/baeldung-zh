## **2。技术和思考**

#### **[> >斩断对代码的心灵感应](https://web.archive.org/web/20220121025305/http://www.daedtech.com/cutting-down-on-code-telepathy)**

**“带外”与 API** 交互所需的知识通常不好。REST 架构通过 HATEOAS 约束和媒体类型解决了这个问题。然而，这不仅仅是 REST 的问题，而是在设计任何 API 时都要问的问题:

`Does the caller require any out of band knowledge to consume the API?`

如果是，这是一个成本；视情况而定，这可能是你想要承担的成本，**但它应该始终是一个有意的决定**。

#### **[> >新独立军](https://web.archive.org/web/20220121025305/http://jacquesmattheij.com/the-army-of-the-new-independents)**

工作/就业格局正在快速变化。因此，**如果你已经独立或者正在考虑这个问题**，这是一个很好的阅读材料，可以给你一些背景知识。

**[> >程序员是一条职业道路，谢谢](https://web.archive.org/web/20220121025305/http://www.daedtech.com/programmer-is-a-career-path-thank-you)**
